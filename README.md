```markdown
# Student Registration — Google Sheets backend (Arabic headers + RTL)

التغييرات الجديدة
- أضفت endpoint `GET /api/setup-sheet` يضع رؤوس الأعمدة بالعربي في الصف الأول (A1:O1) ويضبط اتجاه الورقة إلى من اليمين لليسار (RTL).
- استخدم هذا endpoint مرة واحدة بعد النشر لضبط الشيت.

العناوين العربية (الصف الأول — A1:O1)
1. رقم الطالب
2. تاريخ الإرسال
3. الاسم الكامل
4. رقم التليفون
5. واتساب
6. رقم ولي الأمر
7. المرحلة
8. السنة
9. المادة
10. المدرس
11. المجموعة
12. المواعيد
13. حالة الدفع
14. الحالة
15. سبب الرفض/ملاحظات

كيفية الاستخدام
1. تأكد أن لديك متغيرات البيئة مضبوطة:
   - GOOGLE_SERVICE_ACCOUNT_KEY : محتوى JSON لمفتاح الـ Service Account
   - SHEET_ID : الـ Spreadsheet ID
   - SHEET_NAME : (اختياري) اسم الورقة، افتراضي "Registrations"

2. انشر على Vercel أو شغّل محلياً:
   - بعد النشر افتح: `https://<your-deploy>.vercel.app/api/setup-sheet`
   - أو نفّذ:
     curl "https://<your-deploy>.vercel.app/api/setup-sheet"

   هذا سيضع رؤوس الأعمدة بالعربية ويحول اتجاه الورقة إلى RTL.

3. بعد التشغيل يمكنك اختبار الفورم عبر:
   POST إلى `/api/submit` أو استخدم واجهة form على الصفحة الرئيسية.

ملاحظات أمان
- endpoint `setup-sheet` يغيّر محتوى الصف الأول لذلك يُفضل تشغيله مرة واحدة ثم حذفه أو حمايته (مثلاً: إضافة قيمة سريّة في env وتحقق منها داخل الكود).
- احرص على عدم نشر مفتاح الـ Service Account في الريبو (ضعه في environment variables في Vercel أو .env.local عند التطوير).

تحسينات مستقبلية مقترحة
- حماية endpoint ب secret token في env.
- واجهة إدارة تُتيح قبول/رفض الصفوف وتحديث الحالة دون تحرير الشيت يدوياً.
- نقل الصفوف المقبولة إلى شيت "المقبولين" تلقائياً.
```
